Motion Tokens

Standardized duration and easing tokens used across all animations.

Duration Tokens

Token Value Use Case Preview
--duration-instant 75ms Instant feedback, opacity toggles
--duration-fast 150ms Button states, small transitions
--duration-normal 200ms Default for most transitions
--duration-smooth 300ms Expanding panels, modals
--duration-slow 500ms Page transitions, complex reveals
--duration-dramatic 1000ms Hero animations, onboarding

Easing Tokens

ease-out Default
Best for elements entering the screen
ease-in-out Movement
Best for movement between positions
cubic-bezier(0.34,1.56,0.64,1)
Spring/bounce for playful interactions
linear Looping
Best for continuous loops (spinners)

Reduced Motion

Respecting User Preferences

Some users have vestibular disorders or motion sensitivities. Always wrap non-essential animations in a prefers-reduced-motion media query. 

/* Remove non-essential animations */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}

/* Or target specific animations */
@media (prefers-reduced-motion: reduce) {
  .animate-fade-in { animation: none; opacity: 1; }
  .animate-bounce  { animation: none; }
  .animate-pulse   { animation: none; }
}

Guidelines

  • Essential animations (loading spinners, progress bars) can remain but should be simplified.
  • Decorative animations (entrance effects, hover lifts) should be disabled entirely.
  • Transitions should use opacity cross-fades instead of spatial movement.
  • Use the toggle at the top of this page to preview how your UI behaves with reduced motion.
  • Test with your OS accessibility settings: macOS System Settings > Accessibility > Display > Reduce Motion.
