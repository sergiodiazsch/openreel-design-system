Do's and Don'ts

Visual guidelines for correct and incorrect usage of OpenReel components.

Buttons

✓ Do

Use the brand color for the single primary CTA. Secondary actions use outlined style.

✗ Don't

Never use multiple primary buttons in the same row. It dilutes the CTA hierarchy.

Colors

✓ Do
Active Error Warning

Use semantic colors for status indicators: green for success, red for error, amber for warning.

✗ Don't
Error Offline

Don't use the brand color for error or warning states. Users won't recognise the severity.

Maintain clear hierarchy: h1 > h2 > body with consistent weight pairing.

✗ Don't

Badges

✓ Do
Completed Failed

Use semantic badge colors: green for success, red for errors. Keep badge count minimal.

✗ Don't
Active HD Urgent Review Pro New

Don't overload a row with too many badges. It overwhelms the user and reduces scannability.

Icons

✓ Do

Keep all icons at a consistent 20x20px size with stroke style.

✗ Don't

Don't mix icon sizes (16, 20, 28) or switch between filled and stroke styles in the same context.

Screen Reader Guidelines

Images & Media
  • Always provide descriptive alt text for images
  • Use alt="" for decorative images
  • Provide captions for video content
  • Describe charts and graphs in text
Icon Buttons
  • Always add aria-label to icon-only buttons
  • Use sr-only class for visually hidden text
  • Avoid title as the only accessible name
  • Group related actions with role="group"

When to Animate

Animate
  • State feedback (hover, active, focus)
  • Entrances and exits (modals, drawers)
  • Loading indicators and skeletons
  • Page transitions between views
  • Micro-interactions (toggles, checkboxes)
Don't Animate
  • Static content layout shifts
  • Error messages (show immediately)
  • Data table updates
  • Anything during initial page paint
  • Excessive decorative motion
Reduced Motion Support

Always respect the user's prefers-reduced-motion setting. Wrap all animations in this media query:

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
