Enhancement Catalog — What Gets Added

Same UI kit, refined details. Every enhancement below is additive — nothing changes, everything improves.

Tier 2 — Motion & Visual Polish

BUTTONS
  • Hover lift: translateY(-1px) + shadow-sm upgrade — subtle elevation on hover
  • Active press: translateY(0) + scale(0.98) — tactile click feedback
  • Focus ring: brand-colored ring with 4px spread, animated opacity in
  • Loading spinner: smooth rotation with opacity fade transition
  • Disabled: gentle opacity transition instead of instant change
INPUTS & FORMS
  • Focus border: border-color transitions smoothly to brand (150ms)
  • Focus shadow: soft brand-colored ring fades in (0 0 0 4px rgba(31,18,222,0.12))
  • Error state: border + ring transition to error-red, gentle shake on invalid submit
  • Placeholder: text opacity animates on focus/blur
  • Toggle/Switch: thumb slides with cubic-bezier, track color transitions
  • Checkbox: checkmark draws in with stroke-dashoffset animation
  • Select dropdown: slides down with opacity (150ms ease-out)
CARDS & SURFACES
  • Card hover: border-color subtly shifts + shadow upgrade from xs to sm (200ms)
  • Interactive cards: translateY(-2px) on hover with shadow-md
  • Modal enter: fade + scale(0.95 -> 1) with backdrop blur transition
  • Modal exit: reverse — scale down + fade out (150ms)
  • Drawer: slide from edge with subtle momentum easing
  • Accordion: height auto-animates with max-height transition
COLORED ACCENT SHADOWS Optional tokens
  • --shadow-brand-sm/md/lg: brand-tinted shadows for primary buttons, CTA cards
  • --shadow-success-sm: green-tinted shadow for success states
  • --shadow-error-sm: red-tinted shadow for error/destructive elements
  • --shadow-warning-sm: amber-tinted shadow for warning states
  • Applied via .or-shadow-brand, .or-shadow-success utility classes
  • Transition on hover — neutral shadow to colored shadow
NAVIGATION & FEEDBACK
  • Tab indicator: underline slides between tabs (not instant jump)
  • Toast enter/exit: slide up + fade in, slide out on dismiss
  • Tooltip: fade + slight translateY on show (100ms)
  • Dropdown menu: scale origin from trigger, fade in (120ms)
  • Progress bar: width transitions smoothly, striped animation on indeterminate
  • Skeleton: shimmer pulse with gradient sweep (existing, refined timing)
  • Badge: scale bounce on count update
GLOBAL
  • prefers-reduced-motion: all animations disabled with single media query
  • Dark mode transition: background + color crossfade (already exists, refined)
  • Motion tokens: --duration-fast: 100ms, --duration-normal: 200ms, --duration-slow: 300ms, --ease-default, --ease-spring

Tier 3 — Advanced Effects (opt-in JS)

  • GSAP ScrollTrigger: components animate in on scroll (fade-up, stagger lists) — data-or-animate="fade-up"
  • GSAP SplitText: heading text reveal, character-by-character — data-or-animate="split-chars"
  • Three.js particle field: ambient floating particles for hero sections — <div data-or-particles>
  • WebGL gradient: animated mesh gradient background — <div data-or-gradient>
  • Parallax depth: layered scroll parallax — data-or-parallax="0.5"
  • SVG morph: icon morphing between states — data-or-morph="icon-a,icon-b"
  • All via CDN, lazy-loaded only when data-or-* attributes detected on page
