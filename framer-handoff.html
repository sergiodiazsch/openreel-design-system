Getting Started

Three steps to use the OpenReel DS in any Framer project.

1

Inject the CSS in Framer Site Settings

Site Settings → Custom Code → Head tag

Open your Framer project, go to Site Settings (gear icon, top right), select the Custom Code tab, and paste the following into the <head> tag area:

<link rel="stylesheet" href="https://openreel-design-system.netlify.app/css/openreel.css" />
<link rel="stylesheet" href="https://openreel-design-system.netlify.app/css/openreel-motion.css" />

This loads all design tokens, component classes, and motion utilities. The motion file is optional but recommended for hover/transition effects.

2

Use code components with the or- class prefix

Insert → Code component → paste HTML

In Framer, use Insert → Code to add a code component. Paste the HTML snippet from the Component Snippets section below. All OpenReel classes use the or- prefix — they are globally available once the CSS is injected.

export default function PrimaryButton() {
  return (
    <button className="or-btn or-btn--primary or-btn--md">
      Get Started
    </button>
  )
}
3

Copy-paste from the DS docs

All 61 components available at index.html

Browse the full component docs to find any component. Each has an HTML tab with copy-ready markup. Paste it into a Framer code component — the injected CSS handles all styling automatically.

Design Tokens in Framer

Once the CSS is injected, all CSS custom properties are available everywhere in Framer — in code components via inline styles, and in the style panel.

Using tokens in Framer code components

// Reference CSS variables directly in style objects
export default function TokenExample() {
  return (
    <div style={{
      backgroundColor: "var(--bg-primary)",
      color: "var(--text-primary)",
      padding: "var(--space-6)",
      borderRadius: "var(--radius-lg)",
      border: "1px solid var(--border-primary)",
      boxShadow: "var(--shadow-md)",
      fontFamily: "var(--font-family)",
      fontSize: "var(--text-base)",
      fontWeight: "var(--font-semibold)",
    }}>
      Token-driven component
    </div>
  )
}

Key token groups

Token Group Examples Use for
--bg-* --bg-primary, --bg-secondary, --bg-tertiary Surface backgrounds, card layers
--text-* --text-primary, --text-secondary Body copy, labels, captions
--border-* --border-primary Card borders, dividers, inputs
--brand-* --brand, --brand-hover, --brand-light CTA buttons, links, focus rings
--text-* (size) --text-xs, --text-sm, --text-base, --text-md, --text-xl Typography sizes
--space-* --space-2, --space-4, --space-6, --space-8 Padding, margin, gap
--radius-* --radius-sm, --radius-md, --radius-lg, --radius-full Border radius on all elements
--shadow-* --shadow-xs, --shadow-sm, --shadow-md, --shadow-lg Elevation for cards, modals, dropdowns
--duration-* --duration-fast, --duration-normal, --duration-slow Transition durations (100ms–300ms)

Tips

Best practices for using the OpenReel DS effectively in Framer prototypes.

Use tokens, not hardcoded values

Always reference var(--token-name) instead of hardcoding hex colors or pixel values. This ensures your prototype automatically adapts when the design system updates, and light/dark mode works for free.

Wrap DS components in Framer override components

Create a Framer override that adds DS class names to native elements. This lets your design team drag-and-drop while still using real production classes. Example: Override a Framer rectangle with or-card to give it proper token-based styling.

Test dark mode in Framer preview

Add a toggle component that calls document.documentElement.classList.toggle('dark') to switch between light and dark mode inside your Framer prototype preview. All DS tokens flip automatically.

Use motion tokens for transitions

When adding hover or state transitions in Framer variants, use var(--duration-normal) (200ms) for standard interactions and var(--ease-spring) for playful animations. This keeps motion consistent with the live product.

Keep prototype and production in sync

Because the prototype uses the same CSS as production, any token change in openreel.css is reflected immediately in both. No manual updates needed — your prototype is always a faithful representation of the live design system.

Avoid mixing DS classes with Framer's generated CSS

Framer may apply its own background/color styles via generated class names. If a DS component looks wrong, wrap it in a zero-padding container div and apply DS classes only to the inner element. Use !important sparingly if Framer styles conflict.
