Login with Inline Validation

A standard sign-in form with real-time field validation, social login options, and accessible error states.

Please enter a valid email address
Password must be at least 8 characters

Best Practices
  • Validate on blur, not on keystroke, to avoid premature error messages
  • Disable submit until all fields pass validation to prevent error flash
  • Use shake animation sparingly -- only for invalid submit attempts
  • Provide social login alternatives to reduce friction
<form novalidate>
  <div class="fi">
    <label for="email">Email</label>
    <input type="email" id="email"
      placeholder="you@company.com" />
    <span class="err-msg">
      Please enter a valid email
    </span>
  </div>

  <div class="fi">
    <label for="pw">Password</label>
    <input type="password" id="pw"
      placeholder="Min 8 characters" />
    <span class="err-msg">
      Password must be at least 8 characters
    </span>
  </div>

  <label>
    <input type="checkbox" /> Remember me
  </label>

  <button type="submit" class="btn-primary"
    disabled>Sign in</button>

Search with Autosuggest

A search input with debounced suggestions, grouped categories, and full keyboard navigation (arrows + enter).

Best Practices
  • Debounce input to avoid excessive rendering (150-300ms)
  • Support full keyboard navigation for accessibility
  • Group results by category for better scannability
  • Show a helpful "no results" state instead of an empty dropdown
<div style="position:relative;">
  <svg class="search-icon" />
  <input type="text" id="search-input"
    placeholder="Search..."
    oninput="handleSearch(this.value)"
    onkeydown="handleSearchKey(event)" />

  <div class="autosuggest-dropdown"
    id="dropdown">
    <!-- JS populates groups + items -->
    <div class="as-group-label">
      Recent Searches
    </div>
    <div class="as-item">
      <div class="as-icon">...</div>
      <span>Result text</span>
    </div>
  </div>
</div>

A filterable interface with chip tags for active filters, result counting, and a collapsible advanced panel.

Best Practices
  • Show active filters as removable chips for visual clarity
  • Always display the result count so users know the impact of filters
  • Provide a "Clear all" action to quickly reset
  • Hide advanced filters behind a toggle to reduce cognitive load
<!-- Filter bar -->
<div class="filter-bar">
  <input placeholder="Filter results..." />
  <select onchange="addFilter('Status', this.value)">
    <option value="">Status</option>
    <option>Active</option>
    ...
  </select>
</div>

<!-- Active filter chips -->
<div id="chips">
  <span class="chip">
    Status: Active
    <button onclick="removeFilter(this)">x</button>
  </span>
</div>

<!-- Results count -->
<span>Showing 24 of 156 results</span>

  Clear all

Contact Form

An accessible contact form with file attachment, character counter, and a success confirmation state.

Drop a file here or browse

PDF, PNG, JPG, ZIP up to 10MB

Message Sent!

Thank you for reaching out. We will get back to you within 24 hours.

Best Practices
  • Use proper label and aria attributes for screen reader accessibility
  • Show character count to set expectations for message length
  • Replace the form with a clear success state on submission
  • Restrict file types and show attached file info with a remove option
<form novalidate>
  <div class="fi">
    <label for="name">Name</label>
    <input type="text" id="name"
      required aria-required="true" />
  </div>

  <div class="fi">
    <label for="message">
      Message <span id="count">0/500</span>
    </label>
    <textarea id="message" rows="5"
      maxlength="500" required></textarea>
  </div>

  <!-- Drop zone -->
  <div class="drop-zone"
    onclick="fileInput.click()">
    <p>Drop a file or browse</p>
    <input type="file" hidden />
  </div>

  <label>
    <input type="checkbox" />
    I agree to the privacy policy
  </label>

  <button type="submit"
    class="btn-primary">Send Message</button>
</form>
